Indentation and Spacing

A bit of research has led me to the following ruleset:

  • Do not place more than one block element on one same line
  • Nest ever in line element inside a block element
  • Line limit is 100 char except for URLS
  • Breaks are not for vertical space they're for paragraphs. Max 1 per row.
  • No whitespace between braces and element names
  • Use h1 tag only once per page for search engine ranking
  • Don't use caps in markup, do it in the CSS instead

CSS intro

CSS stands for Cascading Style Sheet.
The style sheet holds the rule set
Rule sets contain one or more of the following: selector { declaration } or more specifically: selector { property: value; property:value; ... } The rule set can live in one of 3 places:

  1. Inline via elementâ€™s opening tag style attribute: <p style="color: blue; font-size: 18px;">Blue, 18-point text</p>
  2. Header via style tag: <head> <style> h1 { color: blue; } </style> </head>
  3. Separate file via <link> tag in the <head>: <head> <link href="style.css" type="text/css" rel="stylesheet"> </head> Note: href = url type = content type, rel = relationship to html file
  4. Option 3 is easiest to run at scale.

Valid Selectors

  • Element Type
  • Element Type Element Type ... (descendant selector for nested types)
  • Element Type + Class (chain)
  • Multiple Element Types (comma separated)
  • ID
  • Class
This is the difference between an ID and class selector: /* Selects all elements with class="column" */ .column { } /* Selects element with id="first-item" */ #first-item { }

If multiple elements share the same rulsets, we can list them:
h1, h2 { color: red; } If elements within elements need a ruleset, we can chain them together:
/* Select h3 elements with the section-heading class */ h3.section-heading { color: blue; } /* Select elements with the section-heading and button class */ .section-heading.button { cursor: pointer; } If multiple selectors are applied to an element, there is an order of application called Specificity
Specificity refers to how a browser decides which styles to display when there are multiple styles defined that could apply to the same element.
Highest : ID, class, type : Lowest
